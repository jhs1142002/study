[모의해킹]
#취약점
 - 이미 알려진 위협으로부터 손상받을 수 있는 구조적인 문제점

#위험성
 - 취약점과 위협이 결합되어서 손상받을 수 있는 영향의 정도를 뜻함
 - 전통적인 정보보호 3요소(기밀성, 무결성, 가용성)에 미치는 영향의 정도를 파악함

#모의해킹 목적
 - 핵심 비즈니스 즉, 중요한 정보시스템 내에 중요정보 탈취(기밀성 훼손)
 - 핵심 비즈니스 즉, 중요한 정보시스템 내에 중요정보 변조(무결성 훼손)
 - 핵심 비즈니스 즉, 중요한 정보시스템 파괴 (가용성 훼손)
 - 중요한 정보시스템 자산에 대한 훼손 뿐만 아니라 해당 시스템에 접속하는 사용자 훼손도 모두 포함
 - 중요정보 예시) 내부 기술정보, 개인정보, 금융정보, 결제조작 등

#현재 가장 많이 수행하는 모의해킹 대상
1. 웹 해킹
 - 외부 웹 서비스의 경우 방화벽에서 오픈하고 있기 때문에 보다 수월하게 내부망 침투가 용이함
 - 1차적으로 웹 해킹을 통해 웹 서버를 장악할 수 있는 가능성
 - 2차적으로 웹 서비스를 접속하는 일반 사용자 및 권리자를 대상으로 한 공격 가능성
 - 3차적으로 웹 서버 장악을 통한 내부망의 타 시스템을 공격할 수 있는 가능성

2. 모바일 앱 모의해킹
 - Android 기반 어플리케이션
 - iOS 기반 어플리케이션

#장애를 유발해 가용성을 침해하지 않는 범위 내에
1. 웹 취약점 자동 스캐너 사용 시 협의 필요
 1) 네트워크 트랙픽 증가로 인한 서비스 마비 유발 가능성
 2) 게시판의 대량의 게시글이 순간적으로 생성되는 문제 발생 가능성
 3) 데이터베이스의 데이터가 일부 수정되거나 삭제되는 무결성 훼손 가능성

2. 서비스 거부 공격과 관련된 Exploit 코드 주입시 협의 필요
 1) DoS, DDoS, DRDoS 등
 2) Exploit Injection 공격 시도 시 주의 필요

#고객사와 협의한 시간 안에 활동
1. 수행IP 고객사에 전달
2. 수행IP로만 정해진 시간과 기간동안만 수행하는 것이 원칙
3. 수행기간이 초과된 시기에 수행하는 것은 불법적인 해킹시도로 간주됨

#모의해킹 범위 - 시스템 접근 기준
1. 외부 모의해킹
 1) 외부자 관점에서 내부 시스템의 모의해킹 목적 달성 시도
 2) 수행 시 실제 외부의 IP를 사용해서 모의해킹 시도

2. 내부 모의해킹
 1) 내부자 관점에서 접근통제 누락 또는 우회 공격을 통해 모의해킹 목적 달성 시도
 2) 실제로 내부자가 권한이 없는 시스템에 접근 또는 권한상승을 통해 내부정보를 탈취하는 사례가 있기 때문에
 3) 수행 시 고객사 내부 프로젝트룸에서 내부 IP를 사용하여 모의해킹 시도
  - 수행 전 필요에 따라 내부 사이트(e.g 총무지원시스템, 그룹웨어 사이트 등)의 접근권한 등을 요청할 수 있음

#모의해킹 범위 - 수행 관점 기준
1. 체크리스트 기반 진단
 1) 인프라 진단의 경우 설정정보를 확인하고 각 항목별 취약점 진단
 2) 웹의 경우 홈페이지에서 각 항목별 점검방법(점검문자열)을 동적으로 페이지의 반응을 확인하는 과정(동적분석)

2. 모의해킹
 1) 웹 취약점 진단을 통해 식별된 취약점들의 위험성을 확인하는 수준
 e.g. SQL Injection 취약점 발견 시 데이터베이스의 중요정보(개인정보 등) 유출까지 확인하는 과정
 e.g. XSS 취약점 발견 시 사이트를 접속하는 사용자를 악성코드 감염이 가능한지 확인하는 과정
 2) 방법론
  - 시나리오 기반 모의해킹(현재 가장 많이 사용함)
  - 블랙박스 모의해킹(일부 수행하고 경우가 있으나 빈번하게 채택하는 방법론은 아님)
  - Red / Blue / Puple Team 침투테스트(고려사항이 많은 방법론이라 현업 모의해킹 보다는 해킹대회에서 주로 채택)

3. 시나리오 기반 모의해킹
 1) 시나리오를 먼저 구성 후 침투 시작
  - 웹 서비스 취약점을 이용하여 내부망 침투 후 연계된 타 시스템에 접근하여 중요정보를 획득
  - 이메일 첨부파일 공격을 통해 내부망 침투 후 연계된 타 시스템에 접근하여 중요정보를 획득
  - 이동식 저장장치(badusb 등) 공격을 통해 내부망 침투 후 연계된 타 시스템에 접근하여 중요정보를 획득
